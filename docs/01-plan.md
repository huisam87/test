# 프로젝트 계획 및 진행 기록

## 1단계: 문제 정의 및 범위 고정
- 목표 사용자: 스마트폰 사용이 익숙하지 않은 어르신
- 범위 축소: 사진/동영상 송수신 연습만 우선 구현
- 제외: 로그인, 서버, 계정 연동, 실제 카카오톡 제어

## 2단계: 정보 구조/흐름 설계
- 홈 메뉴 4개 설계
  - 사진 보내기
  - 사진 받기
  - 동영상 보내기
  - 동영상 받기
- 각 메뉴당 2개 시나리오로 총 8개 연습 구성

## 3단계: 오프라인 웹앱 구현
- 정적 서버 기반 실행 구성
- 단계형 연습 플레이어 구현
- 로컬 진행 저장(localStorage) 구현
- 서비스워커 캐시 적용

## 4단계: Android APK 래핑
- WebView 앱 생성
- 웹 자산을 android_asset에 포함
- debug APK 빌드 파이프라인 구성

## 5단계: 실행 자동화/검증
- 웹 가상 실행 스크립트 추가
- 에뮬레이터+APK 자동 실행 스크립트 추가
- APK 빌드/설치/실행 검증 완료

## 6단계: 기준 버전 정합화
- 기준을 Android 14 + 카카오톡 10.6.5로 고정
- 시나리오 문구를 카톡 버튼명 중심으로 정리
